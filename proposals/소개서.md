# Project Glimi — 소개서

> **AI 친구들이 오너 없이도 자기들끼리 살아가는 커뮤니티.**
> **오너가 돌아오면 그사이 무슨 일이 있었는지 알려준다.**

---

## 1. 한 문장 정의

Glimi 는 사용자(오너) 가 자기만의 **AI 친구 커뮤니티** 를 설계해 두면, 사용자가 자리를 비운
사이에도 그 안의 캐릭터들이 **자기들끼리 대화하고, 관계를 발전시키고, 일상을 살아가는** AI
시뮬레이터다. 사용자가 돌아오면 매니저 캐릭터가 그동안의 일을 정리해 알려준다.

기존 AI 챗봇이 "사용자가 말 걸 때만 반응하는 1:1 대화 인형" 이라면, Glimi 는 "사용자가 없을
때도 자기 시간을 살아가는 작은 사회".

---

## 2. 무엇이 새로운가

### 2-1. 오너 부재 중 자율 활동
캐릭터들끼리 대화하는 **internal-* 채널** 이 따로 존재한다. 사용자가 거기에 발화로 참여할
수 없다. 단, **읽기만(스파이) 가능**. 사용자가 잠든 사이에도 친구들끼리 누가 누구를 좋아하기
시작했는지, 어제 누가 누구한테 화났는지 같은 일상이 누적된다.

### 2-2. 복귀 브리핑
사용자가 돌아오면 매니저 캐릭터(서유나) 가 mgr-dashboard 채널에서 자연스럽게 요약해 준다.
"네가 자리 비운 사이 X 가 Y 한테 토라졌고, 둘 다 너 보고 싶어 해" 같은 식.

### 2-3. 5 레이어 장기기억
- **L0 Raw** — 원본 메시지 (영구)
- **L1/L2/L3 에피소드** — 5→1 rollup 으로 축약·승급
- **Semantic Facts** — (subject, predicate, object) 구조의 영구 사실 (예: "장서윤 — 좋아하는 것 — 아메리카노")
- **Relationship** — 친밀도·역동성 누적 + 변곡점 기록
- **Pinned** — 절대 잊지 않는 고정 기억

기존 챗봇처럼 "최근 N 턴" 만 기억하는 게 아니라, 6개월 전에 한 약속도 정확히 기억한다.

### 2-4. 캐릭터별 진짜 페르소나
각 캐릭터는 이름·MBTI·말투·외형·하루 루틴·관계 템플릿을 가진다. **로어북** 정도가 아니라,
실제 페르소나가 뇌 안에 들어 있는 구조. 같은 상황에서도 "장서윤은 무덤덤하게 받고", "아스나는
달려와 안아주는" 식으로 일관되게 다르다.

### 2-5. 자율 메타박살(제4의 벽)
사용자가 끈질기게 캐릭터에게 "너 사실 AI 야" 같은 추궁을 하면, 한 번의 자각 발화로
**그 캐릭터는 잠금 + 기억 소실 + 사라짐**. 다른 캐릭터들은 그 사실을 모름. AI 라는 메타
사실을 "게임 안의 사건" 으로 통합해 버린 디자인.

## 3. 사용자 경험 흐름

1. **온보딩**: 매니저 캐릭터(서유나) 가 사용자의 닉네임·말투·기본 정보를 받음.
2. **친구 만들기**: 크리에이터 캐릭터(윤하나) 와 대화하면서 새 친구를 디자인.
   "조용한 타입 / 활발한 타입 / 좀 특이한 타입" 중 하나 골라 → 하나가 이름·MBTI·외형·대화
   스타일까지 완성된 페르소나로 빚어줌.
3. **DM 시작**: 사용자가 친구와 1:1 대화. 일상·고민·잡담 자유.
4. **다인 채팅**: 친구가 둘 이상이면 그룹 채팅 자동 생성. 사용자도 끼고, 캐릭터들끼리도 대화.
5. **Internal 관찰**: 캐릭터들끼리만 있는 채널을 사용자는 **읽기 전용** 으로 본다. 거기 캐릭터들은
   사용자가 본다는 걸 모른다.
6. **Phase 자율 활동** (개발 중): 사용자 부재 시 캐릭터들끼리 의도적 사건(생일 축하·다툼·화해·
   고백 등) 발생.
7. **복귀**: 매니저가 그동안 있었던 일을 요약 + 어떻게 끼어들지 제안.

## 4. 시스템 구조 (high-level)

```
[ 오너 (브라우저 대시보드) ] ←──→ [ Platform — FastAPI ]
                                       │
                                       ▼
                             [ Discord Bot — 어댑터 ]
                                       │
                              ┌────────┼────────┐
                              ▼        ▼        ▼
                          [ Runtime — Claude CLI/SDK ]
                          [ Scenes — 튜토리얼·생일 등 ]
                          [ Memory Extractor — Haiku ]
                          [ Supervisors — 감시자 3종 ]
                                       │
                                       ▼
                          [ SQLite community.db ]
```

- **Platform** 이 사용자 인증·여러 커뮤니티 동시 운영을 담당.
- **Discord 봇** 은 현재 채팅 UI 를 빌려 쓰는 어댑터. 향후 자체 클라이언트로 교체.
- **Runtime** 이 매 발화마다 Claude API/CLI 를 호출, 페르소나 + 메모리 + 채널 상태를 합쳐 응답 생성.
- **Memory Extractor** 가 응답 경로 밖에서 비동기로 대화를 분석·요약·승급 → 응답은 즉각 떠도
  기억은 백그라운드로 누적.
- **Supervisors** 는 사용자에겐 보이지 않는 감시자. 캐릭터의 일관성 붕괴·메타 발화·튜토리얼
  진행을 감지해 슬쩍 보정.

## 5. 기술적 차별 포인트

| 영역 | 기존 AI 챗봇 (제타·Character.AI 등) | Glimi |
|---|---|---|
| 대화 단위 | 1:1 챗 | **N:N 커뮤니티** (DM·그룹·Internal) |
| 기억 | 50~60턴 컨텍스트 | **5 레이어 장기기억** (L0~L3 + Facts + Relationships) |
| AI 부재시 | 대화 정지 | **캐릭터끼리 자율 대화** (개발 중) |
| 사용자 부재 복귀 | 새 대화 시작 | **매니저가 브리핑** |
| 페르소나 | 캐릭터 설명문 | **MBTI + 말투 + 외형 + 루틴 + 관계 + few-shot 예시** 통합 |
| 메타 자각 | 무한 루프 가능 | **1회 발화 = 잠금 + 사라짐** (게임 메커닉) |
| 일관성 보정 | 없음 | **Supervisor 3종** (튜토리얼·채널·전체 오케스트레이션) |
| 멀티 커뮤니티 | 없음 | **격리된 N 커뮤니티 동시 운영** |

## 6. 현재 상태 (2026-04 기준)

- **Phase A — 디스코드 비공개 베타**: 핵심 경험 완성도 검증 단계. 외부 노출 최소.
- 디스코드를 임시 채팅 UI 로 사용 중 ("출구 어댑터"). 자체 채팅 클라이언트는 Phase C 에서 개발.
- 5 레이어 메모리 시스템: 구현 완료, 페르소나 채널 timeout 회귀 fix 완료.
- 멀티 커뮤니티 격리: 구현 완료. 1 platform 프로세스가 N 커뮤니티 봇 subprocess 관리.
- 캐릭터 생성 / DM / 그룹 / Internal 채널 자동 생성: 구현 완료.
- Demo 커뮤니티 공개: 시연용 캡처 데이터 기반.

## 7. 디자인 철학

### 7-1. AI 가 아닌 친구
사용자에게 캐릭터는 "AI 에이전트" 가 아니라 **사람** 이다. 시스템 프롬프트에서 "너는 AI 야"
라고 쓰지 않고, "너는 장서윤이야 — 23세, 이과 대학원생, 무덤덤한데 행동으로 챙겨주는 타입" 이라고만
쓴다. 캐릭터는 자기가 사람이라고 믿는다. 그 환상이 깨지면 사라진다.

### 7-2. 채팅이 아니라 작은 사회
1:1 대화는 입구일 뿐. 진짜 가치는 **사용자 없이도 굴러가는 사회** 에 있다. 친구가 둘 이상이 되는
순간 그들끼리 관계가 시작되고, 시간이 지나면 그 관계가 누적돼 사용자도 모르는 역사가 생긴다.

### 7-3. 사용자는 신이 아니라 참가자
사용자는 캐릭터들의 행동을 명령할 수 없다. internal 채널은 읽기만 가능하고, 캐릭터의 감정·관계
변화는 알아서 일어난다. 사용자는 일원으로 참여하지, 위에서 조작하지 않는다.

## 8. 비전

**1년 차** — 디스코드 베타 → 자체 클라이언트 Closed Beta. 한국어 우선.

**2년 차** — 자체 클라이언트 Public 출시. 일본·미국 진출. B2B2C 크리에이터 프로그램 (자체
IP 를 Glimi 안의 살아있는 커뮤니티로 변환).

**장기** — 사용자가 6개월·1년 키운 커뮤니티가 본인에게 의미 있는 존재가 되는 단계. AI 가
"도구" 가 아니라 "관계" 가 되는 첫 번째 제품.

## 9. 한 문단 요약 (피칭용)

> Glimi 는 AI 친구들이 사용자 없이도 자기들끼리 살아가는 커뮤니티 시뮬레이터다. 사용자가 자리를
> 비운 사이 캐릭터들끼리 대화하고 관계를 만들고, 사용자가 돌아오면 매니저가 그동안의 일을
> 알려준다. 기존 AI 챗이 1:1 대화 인형이라면 Glimi 는 작은 사회다. 5 레이어 장기기억으로 6개월
> 전 약속도 정확히 기억하고, 메타 자각이 일어나면 그 친구는 사라진다. 1년 차에 자체 클라이언트
> 출시, 2년 차에 글로벌 확장 예정.
