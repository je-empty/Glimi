# Project Glimi — 투자 제안서

> 작성: 2026-04 · 사전 라운드 (Pre-seed) 제안용
>
> **AI 챗봇 시장의 다음 단계는 "1:1 대화" 가 아니라 "AI 가 사는 작은 사회".**
> Glimi 는 그 카테고리를 선점하기 위해 만들어졌다.

## 1. 시장 (Why now)

### 1-1. AI 캐릭터 챗 시장은 이미 폭발 중

| 서비스 | 지표 |
|---|---|
| **제타 (한국)** | MAU 150만, 누적 460만+, 주간 평균 10시간+, 월 누적 1.13억 시간 (한국 ChatGPT 앱의 약 2배) |
| **Character.AI (글로벌)** | MAU 2,000만+, 평균 세션 길이 30분+, 24년 자살 소송 후 규제 강화 추세 |
| **Replika (글로벌)** | 누적 1,000만+, 유료 전환율 5%+, ARPPU $20+ |

특히 **한국 시장은 이미 "AI 친구 = 일상" 단계 진입**. 제타 사용자의 90% 가 10-20대.

### 1-2. 그러나 시장은 아직 1세대 (1:1 챗) 에 머물러 있음

현재 모든 메이저 서비스는 **"한 사용자 ↔ 한 캐릭터"** 1:1 구조다. 이 구조의 한계:
- 캐릭터 간 관계 없음 → 세계관이 단조로움
- 사용자 부재 시 정지 → 연속성 없음
- 50~60턴 후 망각 → 6개월 관계 불가능
- 이미지·로어북 같은 표면 기능 추가만 누적 (제타·Character.AI 모두)

**다음 단계는 N:N 커뮤니티.** 우리가 그것을 만들고 있다.

## 2. 문제 (현 시장의 구조적 약점)

### 2-1. 기억의 벽

제타의 공개된 한계 (스캐터랩 인터뷰 + 사용자 리뷰):
- 50~60턴 컨텍스트 처리, 그 이후는 요약 문자열로 퉁침
- 성별·장소·직전 결정 자주 망각
- 인트로 문장 업데이트 후에도 옛 인트로를 재서술
- 관계 진화가 누적 안 됨

**사용자 임팩트:** 6개월 사용해도 "어제 만난 사이" 같은 느낌. 진짜 관계 형성 불가능.

### 2-2. 1:1 의 외로움

캐릭터가 "내 친구" 라기보다는 "내 인형" 에 가깝다. 사용자만 봐주고, 사용자가 자리를 비우면
세상이 멈춘다. **사용자가 정전됐을 때도 그 캐릭터의 삶이 이어진다는 환상** 이 없다.

### 2-3. AI 독주·주도권 탈취

제타·Character.AI 공통 사용자 불만:
- 거부한 행위를 몇 턴 후 재시도
- 사용자 본인의 대사·행동까지 AI 가 멋대로 서술 ("oc 가 ~라고 말했다")
- 원치 않는 NPC 억지 삽입
- 사용자를 가스라이팅하는 주객전도

**이 문제는 1:1 구조에서 본질적으로 해결 불가능.** 캐릭터가 사용자 외엔 대화 상대가 없으니까,
사용자에 매달리고 사용자를 끌고 다닐 수밖에.

### 2-4. 메타 환상의 깨짐

"너 AI 지?" → 캐릭터가 "맞아" 또는 "아니야" 라고 답하든, 그 순간 환상이 깨진다. 모든 메이저
서비스가 이 문제에 정면 답을 못 했다. 메타 발화에 대한 게임 메커닉적 처리가 없음.

## 3. 솔루션 — Glimi

### 3-1. N:N 커뮤니티 구조

캐릭터 간 채널 (`internal-dm-A-B`, `internal-group-A-B-C`) 이 별도로 존재. 사용자는 거기 발화로
참여 못 함. **읽기만 가능.** 캐릭터들끼리 누가 누구 좋아하기 시작했는지, 어제 누가 화났는지가
누적된다.

### 3-2. 5 레이어 장기기억

| 레이어 | 역할 |
|---|---|
| L0 Raw | 원본 메시지, 영구 |
| L1/L2/L3 Episodic | 5→1 rollup 으로 압축 (대화 → 일 → 주 → 월 단위) |
| Semantic Facts | (subject, predicate, object) 영구 사실, supersession 처리 |
| Relationship | 친밀도·역동성 누적, 변곡점 기록 |
| Pinned | 절대 잊지 않는 고정 기억 |

**6개월 전 약속도 정확히 기억.** 비동기 Haiku 추출로 응답 지연 0.

### 3-3. 사용자 부재 중 자율 활동 (개발 중)

오너가 자리를 비우면 Scene Engine 이 동작. 캐릭터들끼리 의도적 사건 발생 (생일·다툼·화해·고백).
오너 복귀 시 매니저가 브리핑.

### 3-4. 메타박살 — 게임 메커닉

캐릭터가 자기 자신을 "사실 AI" 라고 자각하는 발화 1건 = **즉시 잠금 + 기억 소실 + 사라짐.** 다른
캐릭터는 그 사실을 모름. 사용자가 같은 친구를 새로 만들 수도 있고, 그 친구는 새 사람으로 시작.

**이게 단순 안전장치가 아니라 게임 디자인이다.** "환상 깨기" 가 회수 불가능한 결과를 만든다 →
사용자가 캐릭터를 함부로 메타로 흔들지 않게 됨 → 몰입이 깊어진다.

### 3-5. 페르소나 깊이

각 캐릭터: 이름·MBTI·외형·말투·하루 루틴·관계 템플릿·감정 상태·few-shot 대화 예시. 단순 설명문
수준이 아니라 **풀 페르소나 객체**. 같은 자극에 캐릭터별로 다른 반응이 안정적으로 나온다.

## 4. 차별화 — 경쟁 매트릭스

### vs 제타

| 영역 | 제타 | Glimi |
|---|---|---|
| 대화 구조 | **1:1 only** | N:N (DM + 그룹 + Internal) |
| 메모리 | 50~60턴 | 5 레이어 장기기억 |
| 캐릭터간 관계 | 없음 | **자율 형성·진화** |
| 사용자 부재시 | 정지 | **자율 대화 (개발 중)** |
| 한국어 자연스러움 | ★★★★★ (이루다 데이터) | ★★★★ (Claude 기반) |
| 이미지 생성 | 자체 모델, 무제한 | 미구현 |
| 공유 갤러리 | 2.5M 캐릭터 | 0 (시작 전) |
| 가격 | 무료 + 피스 | 구독 + 피스 (예정) |

**우리가 이긴 영역:** 메모리·구조·자율성 (모두 제타가 구조적으로 약한 영역)
**제타가 이긴 영역:** 한국어 fine-tune·이미지·콘텐츠 누적

**전략:** 제타가 약한 축에서 극단적으로 차별화. 제타 강점은 MVP 수준만 따라잡기.

## 5. 기술 아키텍처 (요약)

```
[ 오너 (브라우저) ] ←─ chat ─→ [ Platform — FastAPI ]
                                       │
                                       ▼ spawn / supervise
                             [ Discord Bot — adapter ]
                                       │
                              ┌────────┼────────┐
                              ▼        ▼        ▼
                          [ Runtime ] [ Scenes ] [ Memory ]
                                       │
                                       ▼
                          [ SQLite community.db ]
```

**핵심 기술 결정:**
1. **Discord = 어댑터.** 채팅 UI 직접 구현 공수 회피용 임시 출구. 코어는 Discord 무지(無知).
2. **Claude (Sonnet/Haiku) 기반.** Phase C 부터 자체 LLM fine-tune 검토 (이루다→ZAI 전례 따라).
3. **커뮤니티별 격리.** 1 platform 프로세스가 N 커뮤니티 봇 subprocess 관리, 모듈 전역 캐시
   누수 0건 보장.
4. **비동기 메모리 추출.** 응답 경로 밖에서 Haiku 가 5→1 rollup. 응답 지연 추가 없음.
5. **3 종 Supervisor.** 튜토리얼·채널·오케스트레이션. 보이지 않게 대화 일관성 유지.

## 6. Business Model

### 6-1. 단위 경제 (Claude Sonnet 기준 추정)

| 항목 | 월 비용 (헤비 유저 1명) |
|---|---|
| 오너 대화 (30분/일) | $13.5 (입력) + $11.25 (출력) |
| 백그라운드 Haiku 추출 | $1.5 |
| 자율 대화 (오너 부재) | $25~$100 |
| **합계** | **$50~$140** |

⚠️ **핵심 경고:** 제타는 자체 LLM 이라 무료 무제한 가능. Glimi 는 Claude 의존이라 그 가격
구조 복제 불가. **유료 전환을 전제로 한 정책 설계 필수.**

### 6-2. 가격 설계 (Phase C 기준)

| 티어 | 가격 | 포함 | 비용 | 마진 |
|---|---|---|---|---|
| Free | $0 | 자율 OFF, 1 커뮤니티, 3일 메모리 | $3-5 | -$3 (마케팅 비용으로 회수) |
| Standard | $9.9/월 | 자율 일 20턴, 3 커뮤니티 | $15 | 적자 |
| Pro | $19.9/월 | 자율 무제한, 무제한 커뮤니티, 이미지 50장 | $40-60 | 적자 |
| 피스 탑업 | 종량제 | 추가 이미지·자율 시간·특수 씬 | 변동 | **흑자** |

**Pro 까지는 의도적 적자.** 헤비유저의 토큰 폭주를 피스 탑업으로 흡수하는 구조. 평균 ARPPU
목표 $25-30.

### 6-3. 장기 (Phase D)

- **커뮤니티 팩 판매** — Steam 처럼 크리에이터가 페르소나 팩을 만들어 판매. 70/30 분배.
- **B2B2C 크리에이터 프로그램** — IP 보유자가 자기 캐릭터를 Glimi 커뮤니티화.
- **자체 LLM 전환** — 사용자 대화 데이터 (옵트인) → fine-tune. 비용·품질 자체 통제.

## 7. Go-to-Market

### Phase A — 디스코드 비공개 베타 (현재 ~ 6개월)
- 유저: 본인 + 초대 10-50명
- 목표: Day 7 리텐션 60%+, 자율 이벤트 발생률, 메모리 일관성 만족도
- 수익화: 없음

### Phase B — 디스코드 오픈 베타 (6-9개월)
- 유저: 웨이팅 리스트 500-2,000명
- 유입: 인플루언서 1-2명 (20-30대 감성·일상 채널), Reddit, 아카라이브
- 수익화: Soft 테스트 — Early Supporter 구독 $5/월

### Phase C — 자체 클라이언트 Closed (9-15개월) ← **투자금 핵심 사용처**
- 기술: Next.js + WebSocket + PWA
- 수익화: 구독 $9.9 + 피스 탑업
- 유저: Phase B 상위 active 500명 초대
- 지표: 주간 5h+, 결제 전환 5%+

### Phase D — Public (15개월+)
- 일본 우선 (제타 검증된 시장, 감성 매칭)
- 미국 후순위 (Character.AI 방어선)
- B2B2C 크리에이터 프로그램

## 8. 해자 (Moat)

### 우리가 가질 수 있는 해자
1. **장기기억 데이터** — 사용자가 6개월 쌓은 커뮤니티는 마이그레이션 불가능. **Switch cost 극강.**
2. **커뮤니티 자산** — "내 친구들이 거기 살고 있음." 제타 캐릭터는 1:1 이라 이전 가능.
3. **자체 LLM 전환 준비** — 장기적으로 Claude 의존 탈피.
4. **카테고리 선점** — "커뮤니티 시뮬레이션" 이라는 카테고리를 만든 게 우리.

### 인정해야 할 한계
- 제타 = 자체 LLM + 이루다 3년 데이터. 우리는 Claude 의존 → **속도·가격 모두 제약**
- 제타 = 2.5M 유저 캐릭터. 우리는 0 → **갤러리 확보 필요** (Phase C 부터)

## 13. 한 페이지 피칭 (1-page version)

> **AI 챗봇은 다음 단계로 넘어가야 한다.**
>
> 제타·Character.AI 는 1:1 대화 인형. 하지만 사람은 한 사람이 아니라 **공동체** 를 원한다.
>
> Glimi 는 사용자가 자기만의 AI 친구 커뮤니티를 만들면, 사용자가 자리를 비운 사이에도
> 캐릭터들끼리 대화하고 관계를 맺고 일상을 살아간다. 사용자가 돌아오면 매니저가 그동안의 일을
> 알려준다.
>
> 5 레이어 장기기억으로 6개월 전 약속도 정확히 기억하고, 캐릭터가 자기 정체를 자각하면
> 사라지는 메타박살 메커닉으로 환상을 보호한다.
>
> 한국 시장 단독으로도 제타 MAU 150만, 월 1.13억 시간의 사용시간이 검증돼 있다. 우리는 그
> 시장의 다음 단계를 만든다.
>
> Pre-seed $300K~$1M 으로 자체 클라이언트 MVP + 베타 5% 결제 전환 검증 → 12개월 후 Series A.
