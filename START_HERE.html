1이 프로젝트가 뭔지

한 줄: Glimi 는 살아있는 멀티 에이전트 하네스다. LLM 호출을 8 레이어로 감싸 "오너가 타이핑을 멈춰도 침묵하지 않는" 에이전트 인구를 만든다.

왜 만들어졌나

LLM 은 본질적으로 request-response. 프롬프트 → 응답. 끝. 혼자서 깨어나지 않고, 후속도 안 하고, 먼저 말 걸지도 않는다. 몇 개를 방에 넣어두면 오너가 타이핑 멈추는 순간 방은 조용해진다.

Glimi 는 LLM 호출 하나를 7 reactive 레이어 로 감싸 응답 품질을 잡고, 그 전체를 proactive supervisor 레이어 로 둘러싼다. Supervisor 는 자체 타이머로 돌며 에이전트의 내면 생각 처럼 nudge 를 주입한다.

  • Reactive 는 이미 있는 대화를 다듬는다.
  • Proactive 는 없던 대화를 시작한다.

대부분의 멀티 에이전트 프레임워크는 1번밖에 없다. Glimi 는 2번을 추가했다.

30초 데모 시나리오 (Community)

14:02 — 너가 잠든 사이, OrchestratorSupervisor 가 3분 타이머에 깨어남
        Haiku judge: "A 와 B 는 1.2h idle, 친밀도 30. 페어 후보."
        → internal-dm-A-B 채널 자동 개설, context="가볍게 근황 토크"

14:03 — A 가 internal-dm-A-B 에서 먼저 말 건다 (오너 모름)
        A: "야 B, 너 요즘 뭐하고 지내?"

14:30 — 너가 dm-B 에 들어가 "뭐해?" 물음
        B: "업무 마감중이야, 방금 A 랑도 얘기했어 ㅋㅋ"
        (방금 대화가 B 메모리에 L1 summary 로 들어가 있음. 친밀도 31.)

이게 Glimi 의 핵심 가치 — 네가 자리를 비워도 무언가 일어난다.

새 코드 추가할 때 첫 질문

이건 Core 냐 Community 이냐?

  • Core = 도메인 무관. "리서치 두 명이 짝지어 토픽 분석" 시나리오에도 똑같이 적용되나? → YES 면 Core
  • Community = 도메인 종속. "유나가 친구 셋 만들기 튜토리얼" 같은 community-sim 메타가 필요 → Community
  • 모호하면 Community 쪽에 두고, 패턴이 굳어지면 Core 로 승격

6Claude Code 로 개발

이 프로젝트는 Claude Code 친화적으로 설계됨. 클론 → Claude Code 실행 → 자동으로 모든 컨텍스트 로드됨.

자동 로드되는 컨텍스트

이 repo 에서 Claude Code 를 실행하면 다음이 자동 주입됨:

  • CLAUDE.md (루트) — 아키텍처 원칙, 작업 규칙, do/don't, 브랜치 전략
  • 각 subdirectory 의 CLAUDE.md 가 있다면 그것도 (예: src/glimi_imagegen/CLAUDE.md)

즉 너가 별도 설명 안 해도 Claude 가 "Discord = 어댑터", "AI co-author trailer 금지", "타임스탬프 UTC-aware" 같은 규칙을 이미 알고 시작.

Claude Code 로 작업하는 워크플로우

  1. repo 클론 + Claude Code 실행 
    git clone https://github.com/jaebinsim/Glimi.git
cd Glimi
claude                                  # Claude Code CLI 실행
  2. 첫 명령으로 컨텍스트 확인 
    이 프로젝트가 뭐고 무엇부터 하면 좋을지 START_HERE.html 기반으로 정리해줘

    → Claude 가 CLAUDE.md + 이 HTML 을 읽고 요약. 진행 방향 sync.

  3. 새 task 마다 새 브랜치 
    develop 에서 feat/ollama-backend 브랜치 새로 파줘. 그리고 task 시작

    → Claude 가 자동으로 git checkout -b feat/ollama-backend

  4. 여러 단계 작업이면 TodoWrite / TaskCreate 요청 
    이 작업 단계별로 todo 만들어서 진행

    → 진행 상황이 사용자에게도 보이고, 컨텍스트 손실 복구 쉬움.

  5. 커밋은 명시적으로 요청

    Claude Code 는 기본적으로 자동 커밋 안 함. "지금 커밋해" 라고 명시해야 커밋. 메시지는 짧게 (1줄 제목).

  6. PR 도 명시적으로 
    이 브랜치 develop 에 PR 올려줘

Claude Code 에게 절대 시키면 안 되는 것

✓ Do

  • "develop 에서 새 브랜치 파줘" 명시
  • "이 task 완료 후 커밋해" 명시
  • 여러 단계면 "todo 만들어서" 요청
  • 막히면 "왜 이게 안 되는지 분석해" 부터

✗ Don't

  • "main 에 바로 push 해" — 절대 금지
  • "커밋 메시지에 Claude 표시 넣어" — AI co-author 금지
  • "--no-verify 로 우회해" — 훅 실패하면 원인 fix
  • "한 번에 다 해" — 검증 안 된 대량 변경은 위험

💡 팁 — CLAUDE.md 가 알아서 막아준다

이 repo 의 CLAUDE.md 에 "AI co-author 금지", "main 직접 작업 금지", "--no-verify 금지" 같은 규칙이 명시되어 있어서 Claude Code 가 이 규칙을 알고 시작함. 별도 매번 알려줄 필요 없음.

7브랜치 전략 — main + develop 만

간단한 두 영구 브랜치. feature/fix 는 한시적 브랜치로 분기 → develop 으로 PR.

main ← develop ← feat/<name> 새 작업 분기 / PR base = develop / 메인테이너가 안정화 후 develop → main 머지
브랜치역할누가 push 하나
main 안정판. 외부 사용자가 클론하면 기본으로 받는 브랜치 메인테이너 전용 · 직접 push 금지
develop working 브랜치. 모든 일반 작업의 통합 지점 메인테이너 / PR 머지로만
feat/<name> 기능 개발 한시적 브랜치 기여자 자유 · PR base = develop
fix/<name> 버그 fix 한시적 브랜치 기여자 자유 · PR base = develop
docs/<name> 문서 갱신 한시적 브랜치 기여자 자유 · PR base = develop
refactor/<name> 리팩터링 한시적 브랜치 기여자 자유 · PR base = develop

금지 사항

  • 금지 main 에 직접 커밋 / push
  • 금지 main 으로 PR (모든 PR 은 develop 대상)
  • 금지 한시적 브랜치를 길게 살려둠 (머지되면 즉시 삭제)
  • 금지 claude/* 류 자동 브랜치 (과거 도구 잔재, 2026-05-17 정리됨)

9코드 규칙 — 한번에 회귀 만들기 좋은 것들

대부분 CLAUDE.md 에서 가져옴. 처음 만지면 가장 쉽게 회귀 나는 항목.

아키텍처 레이어링

✓ Do

  • 새 기능: "이 로직을 Telegram·웹챗 어댑터에서 재사용 가능한가?" 통과해야 Core 자격
  • 플랫폼 중립 타입만 사용 (str, int, dataclass)
  • Discord 결합 코드는 src/bot/ 안에 (또는 향후 apps/community/)

✗ Don't

  • src/core/* 에서 import discord — 절대 금지
  • Discord 타입이 코어 함수 시그니처에 새는 것
  • src/core/prompts/ 에서 src/bot/ import
  • 예외: src/core/sync.py 는 과도기적으로 OK

프롬프트 작성

✓ Do

  • 위치: 범용 = src/core/prompts/en/ · scene 전용 = src/scenes/{scene}/
  • 영어 정본. 한국 특화 (ㅇㅇ / 카톡) 는 locale.py helper
  • 모델 dialect 는 model.tool_call_syntax_hint() helper

✗ Don't

  • <tools> 같은 syntax 하드코딩 (provider 마다 다름)
  • 레거시 [CMD:...] / [ACTION:...] / [QUERY:...] — 전수 제거됨
  • 한국어 f-string 안에 {한글_변수} placeholder — NameError 주의

사용자 보이는 텍스트 (UX)

금지 메타 용어 노출

사용자가 보는 텍스트에서 "에이전트", "멤버", "봇", "AI" 등 메타 용어 절대 X. 시스템 프롬프트에선 "다른 사람은 이름/친구들" 같은 자연스러운 표현. <tools> 블록은 mgr-system-log 에만 노출.

상태 / 캐시

  • 메모리·감정은 system prompt 에 안 넣음 — agent_runtime 이 user prompt 에 채널별 동적 주입
  • 프로필 수정 후 invalidate_cache() + runtime.refresh_agent() 쌍으로 호출
  • 타임스탬프 = UTC-aware ISO (src.core.timeutil.now_utc_iso()) — SQLite CURRENT_TIMESTAMP 직접 사용 금지
  • dm- / mgr- 채널 삭제 금지 (보호됨)
  • 루트에 봇 토큰·커뮤니티 DB·런타임 상태 X (dev/test_defaults.json 만 예외)
